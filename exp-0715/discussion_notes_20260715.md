## 1. misaligned_mass(错位质量)与 agree(对半一致率)是什么

这两个是 `targeted` 模式每 10 步记录的自检指标,监控方法唯一的新原料——逐 token 影响力符号。

### 1.1 错位质量 misaligned_mass

每个生成 token 上有两个量:

- **d_t(教师信号)**= log π_教师(y_t) − log π_学生(y_t):教师比学生多喜欢这个 token 多少,符号是教师想推的方向;
- **ι_t(影响力)**:估计出的"强化这个 token 会让正确率升还是降"。

两者同号(d_t·ι_t > 0)= 教师与验证器方向一致;异号 = **错位**(教师使劲推的方向按估计反而降低正确率)。每个 token 带质量 |d_t·ι_t|,则

> misaligned_mass = 错位 token 的质量和 ÷ 全部 token 的质量和 ∈ [0, 1]。

例:三个 token 的 |d·ι| 为 5、3、2,其中 5 那个错位 → misaligned_mass = 5/10 = 0.5。

**为什么关键**:论文推论 1 说 targeted 每步相对 OPD 的可证优势 = **2 × 错位质量**。实测 ≈ 0.46 有两种互斥解读:

1. 符号是对的 → 教师近半信号在帮倒忙,翻转收益巨大 → targeted 应大幅赢 OPD;
2. 符号是纯噪声 → 每个 token 随机抛硬币,约一半质量随机落在"错位"侧 → 该值恰好钉在 0.5 附近。

实际 targeted 并没有赢 OPD ⇒ 证据指向解读 2(符号≈噪声,即诊断 F1)。

### 1.2 对半一致率 agree

经典分半信度(split-half reliability):把一批采样拆成两半,各自独立估每个 token 的影响力符号,agree = 两半在有效 token 上的一致比例。

- 纯噪声 → 0.5;完全稳定 → 1.0;实测 ≈ 0.76。
- 若两半各以概率 p 命中真符号且相互独立,期望一致率 = p² + (1−p)²;由 0.76 反解 p ≈ 0.86。
- **但独立性不成立**(见 §2.3),0.76 是被抬高的上界,真实 p 可能明显更低。它从未对照外部真值验证过。

它同时也是训练机制:门控只翻转两票一致的 token,不一致的回退 r_t = d_t(等同 OPD)。

## 2. 分半具体怎么算?state 不同为什么能比

### 2.1 代码机制(gsm8k.py `compute_influence`,约 649 行起)

1. 每组 G=8 条 rollout 按组内编号奇偶分半(`half_f = (bi % group_size) % 2`);
2. 一遍扫描同时累加三个矩阵:全批 **G**、半批 A 的 **G_A**、半批 B 的 **G_B**,每个都是
   G = Σᵢ Aᵢ Σₛ (e_{y_s} − π_s) h_sᵀ ∈ R^{V×h}(陪审团所有 token 贡献之和);
3. 给某条 rollout(设属半批 A)的 token 定符号,算两票:
   - **票 1** `iota`:全批 G 减去该 rollout 自己整条贡献(精确留一);
   - **票 2** `iota_alt`:**对面半批** G_B(天然不含自己,无需留一);
4. 门控(gsm8k.py:564):两票同号 → sign(ι̂)|d_t|;否则 → r_t = d_t。

### 2.2 关键澄清:不需要任何人访问相同的 state

影响力是**参数空间(最后一层 unembedding 子空间)里的内积**,不是 state 空间的匹配:

- 陪审团各自用**自己的** (state, token, 优势) 贡献进 G;加总后 G 只是一个 V×h 矩阵——"验证器认为参数该往哪动"的估计,具体 state 已被求和吸收;
- 被打分的 token 带**自己的**隐状态 h_t,算 ι̂(y) = (G h_t)_y − E_π[G h_t];
- 展开后出现 ⟨h_s, h_t⟩:跨 rollout 传递信号的媒介是**表示空间相似度**——别的解答里"类似语境写了类似内容且答对了"的 token 给当前 token 投正票。

这同时解释了:(a) 整批 G vs 按题 G 之争的实质(跨题借力 vs 跨题串扰);(b) 估计天生很噪(陪审团仅 8–16 条、±k/8 加权、经相似度间接传导)。

### 2.3 两票并非独立(0.76 有水分的机制)

票 1 的陪审团 = 全批去己(≈15 条),票 2 的陪审团 = 对面半批(8 条),**后者完整包含于前者**。两票共享一半以上数据,一致率天然偏高。一行代码的对照实验:把票 1 改成"本半批去己"(与票 2 陪审团不相交),看 agree 掉多少——掉幅即相关性水分。终极版即诊断计划 D2/D3:用 256 条大陪审团当参考真值,直接量符号准确率。

## 3. temperature = 1.0 正常吗

### 3.0 先纠正一个概念:τ=1 不是贪心

温度定义:p(y) ∝ exp(z_y/τ)。**τ=1 = 按模型自然分布随机采样;贪心是 τ→0**(实现上写 `temperature=0` 或 `do_sample=False`)。例(两候选原概率 0.7/0.3):

| τ | 选高概率项的概率 |
|---|---|
| 0(贪心) | 100% |
| 0.5 | ≈ 84% |
| **1.0** | **70%(= 原分布)** |
| 2.0 | ≈ 60% |

若 τ=1 是贪心,同题 8 条采样会完全相同、所有组都退化——而日志 nondegen 有 0.67–0.75,可证采样确实在随机。

### 3.1 结论:τ=1 在这套实现里是**唯一自洽**的选择

策略梯度要求采样分布 = 被优化分布。损失里的 logP_S 取自自然分布(τ=1),采样就必须 τ=1;`top_p=1.0` 是同一逻辑的配套(任何截断式采样同样破坏一致性)。同行:DAPO、verl 默认 τ=1;TRL GRPO 默认 0.9 但它把损失侧 logits 也除以同温度——**要么两边都是 1,要么两边一起降,不能只改一边**。现实现没有"两边一起降"的机制,故 τ=1 唯一合法。

### 3.2 副作用(与诊断直接相关)

1. 0.5B 小模型在 τ=1 下爱写长、不收尾 → 可能是 DeepMath 截断率 ~45%(F3)的部分成因。**正解是提 cap 或屏蔽截断样本,不是降温**;
2. 温度控制组内多样性即 nondegen(F4):降温 → 更多全对/全错组 → 信号更少。当前 nondegen 已仅 0.31,降温只会更糟。

评测贪心(τ=0)与训练 τ=1 不矛盾(pass@1 惯例、不参与梯度);d_t 与采样温度无关(定义在自然分布上)。

### 3.3 若将来想动温度

「tempered 自洽」:采样 τ<1 同时损失侧 logP 换成 log-softmax(logits/τ)(优化降温策略),或加重要性权重。可作低优先级消融(τ∈{0.7,1.0} 各跑 GRPO 150 步,看 nondegen/cap/学习速度),排在诊断计划 A–D 之后。

## 4. KL penalty λ=0,要不要加回来?

**结论:不急着默认加回;先把"是否真在漂移"变成可观测事实(零成本),再用小消融决定。**

### 4.1 当时设 0 的依据(合理)

① 论文 §2.2:无参考形式下 leash 梯度在当前点恰为零,λ=0 与论文自洽;② 对标的 G-OPD 论文 harness 就是 KL=0;③ 现代 RLVR 实践(DAPO、LUFFY)流行去 KL。

### 4.2 为什么这些依据在此处打折

**DAPO 们敢去 KL 是因为有别的刹车**(PPO 裁剪、大批量、动态采样、超长过滤)。此实现一个刹车都没有:每批一步梯度(重要性比恒 1,裁剪失效)、每步仅 16–32 条 rollout、GSM8K 版常数 lr 跑 800 步。且论文自洽性是**逐步的**,不约束 800 步累积漂移。证据 = F6:三方法均 ~150 步冲高后回落;targeted 末 18 个记录步 5 个整批全错、恰好跌回基线。(前提:峰值需经复评确认非噪声。)

### 4.3 决策顺序

1. **立刻(零训练)**:把 KL(π_θ‖π_init) 作为**仪表**每步记录(不进 loss),连同熵、长度;跑 R0.1 复评旧检查点确认"峰值真实、后期真掉"。若峰值是噪声 → 问题主要在 F2,加 KL 反而雪上加霜。
2. **若漂移实锤**,按性价比:先**选峰汇报 + 早停**(快照都在,不改动力学);再 **kl_coef ∈ {0, 0.01, 0.04} 消融**(0.04 = DeepSeekMath 经典值,k3 估计器已实现),**锚 = init**。
3. **targeted 模式特别注意**:不要锚 teacher——方法的意义就是在翻转 token 上*推离*教师,teacher 锚会把方法抹平回 OPD;算法 2 已自带对症刹车(tilt_clip=5)。
4. **理论备注**:λ>0 的外部锚使论文逐步证书不再覆盖整个更新(论文原文:外部锚的 leash 是 uncertified trust-region pull)。加 KL 是工程稳定性选择,不是回归论文,写作时要说明。

## 5. 成本定位:targeted 要 G 条 rollout,OPD 的省 token 优势没了,好处在哪?

### 5.1 成本账(每题)

| 方法 | 采样 | 教师前向 | 说明 |
|---|---|---|---|
| OPD | 1 条 | 1 次 | 稠密监督,1 条就够 |
| GRPO | G 条 | 0 | 每条只有 1 比特信号 |
| targeted | G 条 | G 次 | 三者最贵(影响力矩阵乘可忽略) |

**承认:token 效率上 targeted 不和 OPD 竞争。**

### 5.2 正确定位:不是"更贵的 OPD",而是"同预算下信号更密的 GRPO"

- **vs GRPO(同 G 条)**:GRPO 每条榨 1 比特摊到几百 token;targeted 给每个 token 稠密带向信号(教师定力度 × 验证器定方向)。买的是单位 rollout 的信息密度;
- **vs OPD**:OPD 的瓶颈是天花板(Pinsker 带 + 质量盲),不是预算——"越过教师"是 OPD 加多少钱都买不到的。G 倍采样买的是这个。

### 5.3 诚实的 regime 分析(解释当前实验为何读不出优势的第三层原因)

- 当前沙盒(教师 84.8 vs 学生 49.6,差 35pp)是 targeted **最没用**的 regime:模仿空间还很大,OPD 就是对的工具;且题目太难 → ⅔ 组全错 → A≡0 → targeted 自动退化成 **8 倍成本的 OPD**;
- 方法的自然栖息地:**蒸馏后期**(学生逼近教师带、OPD 收敛停下后——论文命题 4 的"OPD 停了它还能走")或**教师有系统性缺陷**(F8 格式不匹配、跨域)。

### 5.4 两个实验设计推论(重要,回应审稿人必问的"你贵 G 倍")

1. **预算对齐的 OPD 基线**:现 harness 让 OPD 也陪跑 G=8,等于没收它省 token 的优势——应加一臂 **OPD@matched-tokens:1 rollout/题 × 8 倍题数**。targeted 赢不了它,实践价值立不住;
2. **混合课程**:先用便宜 OPD(1 条/题)吃完模仿红利到平台期,再切 targeted 当**收尾器**(只在 OPD 停滞且组内有对错混合的题上开验证器机器)。保留大部分 OPD 价格,GRPO 级开销只花在有价值处。

### 5.5 "每题 G=8"的耦合可拆(降本方向)

组采样身兼两职:① 优势基线(A = V − 组均值);② 影响力陪审团 ĝ_Q。其中 **g_Q 是全局向量**(E_x 在外),陪审团可跨题共享、跨步 EMA,定符号可用只推理的大陪审团、训练只回传少数几条;真正刚需组的是基线——换跨题基线(RLOO 式/难度分桶)理论上可把每题压到 1–2 条,代价是基线方差(作为研究方向,不是默认)。

## 7. 补充讨论(2026-07-16):G 矩阵能不能只存"出现过的 token"?

**问**:V×h(151936×896)的 G 矩阵太大,只存采样中出现过的 token 对应的行不行?

**答:直觉对一半;但稠密 V×h 恰恰是最省的表示,且 G 从不写盘(只是每个陪审团在 GPU 上存活几秒的临时缓冲,落盘的都是每 token 标量)。**

把 G 拆开看:

> G = Σ_s a_s e_{y_s} h_sᵀ(**E 部分,稀疏**:只有采到过的 token id 行非零)
> − Σ_s a_s π_s h_sᵀ(**P 部分,稠密**:π_s 是 softmax,对全词表非零)

打分 ι̂(t) = (G h_t)_{y_t} − ⟨π_t, G h_t⟩ 的两项里:第一项确实只查一行(稀疏够用);
**第二项是全词表期望**(中心化项,保证 ι̂ 对 π_t 均值为零,论文式 15 的减项)——π_t 稠密,必须对 G 全部行加权求和。
丢掉它符号整体漂移;本实验审计的就是符号,被审计对象里不能引入近似。

**为什么稠密 V×h 反而是压缩**:h=896 是秩瓶颈——不管陪审团多少 token,信息都被压进秩 ≤896 的 V×h 矩阵。

| 表示 | 大小/代价 |
|---|---|
| 稠密 G(V×h,fp32) | **545 MB**,与陪审团大小无关 |
| 保留陪审团全部 π_s([N_s,V] fp16) | G=8 约 1.0 GB;G=64 约 7.8 GB;参考陪审团(10 万 token)约 30 GB |
| 核形式(不建 G,⟨π_s,π_t⟩ 全对) | 计算量 N_s²·V:G=64 约为建 G 的 14 倍;参考陪审团 ~10¹⁵ FLOPs,不可行 |

陪审团 token 数 ≥ 3200 > h=896,聚合成 G 已经比任何"按 token 存"更小更便宜。训练代码同样如此,探针照搬 = 与训练逐位一致。
实验峰值同时持有 3 个 G(全组+两半)= 1.6 GB,远低于单卡上限,无真实内存压力。

**这个想法成立的两个场景**(记录备用):
1. **top-k 近似 G 是训练侧的合法降本方向**(π_s 截断到 top-k + 尾部修正)——其符号保真度可用本探针加一个
   "exact vs top-k 符号翻转率"对照臂来量(可选扩展,参考真值本身必须精确);
2. 词表更大/卡更小时,第一项走稀疏行、第二项走标量核展开——当前规模下反而更贵,不采用。
