影响力符号探针(D2/D3):设置、计算与判读

这份教程解释 exp-sign-probe/ 里的每一步:实验怎么设置、每个阶段往磁盘里存什么、 每个量用什么公式算出来、最后用哪些预先写好的规则下判断。零训练、纯推理,单卡 4–6 小时。

1这个实验回答什么问题

Targeted-OPD 的全部理论保证都经过一个量:每个 token 的影响力符号 sign(ιt)—— 它说"强化这个 token 会提高还是降低正确率"。训练时这个符号由每题 m=8 条采样组成的"小陪审团"估出来(m 即日志里的 group_size,报告表格中的 G 列)。 复盘发现两个可疑数字:错位质量 ≈ 0.46(纯噪声恰好给 0.50)、对半一致率 ≈ 0.76(两半共享数据,会虚高)。 但符号的真实准确率从来没被测过——这就是诊断计划里的疑点 F1,整个链条上承重的未知数。

本实验用 R=256 条采样组成"大陪审团"造出参考真值,然后:

  • D2:按训练的完整算法,用大小 m ∈ {8, 16, 32, 64} 的小陪审团重估符号,量它们对参考真值的准确率——回答"训练配置下符号到底多准,加大陪审团能买回多少";
  • D3:把训练里的两票(全组留一 vs 对面半批)换成三票对照,量出 0.76 一致率里有多少是共享数据带来的虚高——回答"agree 这个仪表本身可不可信"。

4核心数学:从采样到符号

下面五个式子就是全部计算。记号:一道题采了陪审团 J,其大小记 m = |J|(训练日志、结果报告与仪表盘表格里的 G 列指的就是 m;本教程把字母 G 留给下面的陪审团矩阵 GJ,避免一个符号两用); 第 i 条的判分 Vi ∈ {0,1};token t 处学生的隐状态 ht ∈ ℝ896(末层 norm 后)、全词表分布 πt ∈ ℝ151936、实际采到的 token yt。

4.1 优势(每条 rollout 一个数)

Ai = Vi − meank∈J(Vk)
(1)

组内均值做基线(训练约定,含自身)。整组同对同错 ⇒ 所有 A≡0 ⇒ 退化陪审团,单独计数、不产出符号。

4.2 教师信号(每个 token 一个数)

dt = log πT(yt | st) − log πS(yt | st)
(2)

"教师比学生多喜欢这个 token 多少"。本实验里它不参与符号,只提供质量权重 |d·ι|(§6)。

4.3 陪审团矩阵 GJ(整个陪审团压缩成一个 V×h 矩阵——注意它与大小 m 是两回事)

GJ = Σi∈J Ai Σt∈i (eyt − πt) ht⊤ ∈ ℝV×h
(3)

这是"验证器策略梯度" ĝQ 的最后一层(unembedding)块:答对组(A>0)的 token 把梯度往里加、答错组往外减。 求和后具体来自哪些 state 已被吸收——它只是"参数往哪动能提高正确率"的一个估计。代码:influence.accumulate_G。

4.4 影响力打分(每个 token 与 G 做一次矩阵-向量乘)

ι̂J(t) = (GJ ht)yt − ⟨πt, GJ ht⟩
(4)

即论文式 (15):(G h)y 减去 softmax 均值。为正 = 按陪审团的估计,强化这个 token 会提高正确率。 不同 rollout 不需要访问相同 state——耦合发生在参数空间,展开后是隐状态相似度 ⟨hs, ht⟩ 在传递信用。代码:influence.iota_from_G。逐票的数值走查见 §4.7。

4.5 精确留一(防"自己给自己投票")

给第 j 条 rollout 的 token 打分时,必须把 j 自己从陪审团里去掉,否则每个 token 会继承本条解答的对错符号(论文附录 B 的偏差)。 去掉后恰好等于减一个同 rollout 的 Gram 修正:

corrj(t) = Σs∈j ⟨hs, ht⟩ · [ 𝟙{ys=yt} − πs(yt) − πt(ys) + ⟨πs, πt⟩ ]
(5)
ι̂(−j)J(t) = ι̂J(t) − Aj · corrj(t)
(6)
已验证
式 (6) 不是近似:tests/test_influence_small.py 用暴力法(真的把 rollout j 从陪审团里拿掉、重建 G 再打分)对照, 误差 < 10−4,连同 G 累加、半批相加等 5 项恒等式全部通过。

4.6 分半投票(训练门控的机制)

陪审团按成员位置奇偶分成两半 → 两个半矩阵 G0、G1(优势仍用全陪审团均值,训练约定)。 一个 token 的"对面半批票" = 用不含自己那半的 G 按式 (4) 打分(天然无需留一)。训练时:两票同号才翻转,否则跟随教师。

4.7 具体例子:ι̂ 到底是对"哪条 rollout 的哪个 state"算的

设某道探针题 x,一个陪审团 J 抽中 m=8 条 rollout,判分 V = (1, 0, 0, 1, 0, 0, 0, 0):组内均值 0.25,按式 (1) 优势 A = (+0.75, −0.25, −0.25, +0.75, −0.25, −0.25, −0.25, −0.25)。 现在看一个具体的 token:第 3 条 rollout(答错,A₃ = −0.25)的第 17 个生成位置。 它的 state s = (题目 x + r₃ 已写下的前 16 个 token);学生在此处有隐状态 h ∈ ℝ896、全词表分布 π;实际采到的 token 设为 y = “3”。计算分三步:

  1. 用 8 条 rollout 的全部 ≈3,200 个 token 按式 (3) 累加出 GJ(形状恒为 V×h,与 m 无关);
  2. 留一:按式 (5)(6) 减去 r₃ 自己 ≈400 个 token 的贡献 → 记 GJ(−3)——证据只剩其余 7 条的 ≈2,800 个 token;
  3. 按式 (4):ι̂ = (GJ(−3) h)“3” − ⟨π, GJ(−3) h⟩。得到一个标量,只取它的符号。

它是怎么"跨 rollout"比较的?把第 3 步沿式 (5) 的核恒等式对整个留一陪审团展开:

ι̂(t) = Σs ∈ 其余 7 条的 token Arollout(s) · ⟨hs, ht⟩ · [ 𝟙{ys=yt} − πs(yt) − πt(ys) + ⟨πs, πt⟩ ]
(5′)

读法:其余 7 条 rollout 的每个 token 投一票,票值 = 它所属 rollout 的优势 × 两个 state 的表示相似度 ⟨hs, ht⟩ × 词面一致性修正(方括号)。 例如:答对的 r₁(A=+0.75)在某个隐状态相似的推理位置(⟨h_s,h_t⟩ 大)也写了 “3”(𝟙 项 = 1)→ 正票, "答对的人在类似局面也这么走" ⇒ 强化;答错的 r₅ 在相似局面写了 “3” → 负票。 注意 r₃ 的第 17 位完全可以和 r₇ 的第 213 位发生作用——从不需要对齐位置或匹配 state, 匹配由表示空间的内积自动完成;不同长度、不同解法的 rollout 之间信息因此流得动。

两个顺带的推论:① 估计目标是固定的总体量,不随 m 变——对给定 θ 和 token, ιt = ⟨g_Q(x) 的 unembedding 块, 该 token 的梯度⟩(同一道题条件下的验证器策略梯度)。 随 m 变的只是估计值的尺度:代码沿用训练的"非归一化求和"约定(G 不除以 m), 于是 ι̂sum = m · ι̂mean,其中均值版 ι̂mean 以 sd ∝ 1/√m 收敛到总体量 (基线 V̄→p(x)、留一去自项后偏差为 O(1/m) 二阶)。因为下游只消费符号,而 sign 对正缩放不变, 省掉归一化无伤——但也因此幅度既不该跨 m 比较、也不携带多少可靠性信息(十分位校准所示)。 一个白送的预测:符号判对率 ≈ Φ(√m·|ι|/σ),是 √m 型缓坡——与 D2 曲线"m 每翻倍约 +4–6pp、到 64 仍未饱和"吻合; ② 若这 8 条恰好全对或全错,所有 A≡0,式 (5′) 每票皆零——退化陪审团没有符号可言。

4.8 跨陪审团大小 m 的比较为什么公平:证据侧 vs 考卷侧

先说清陪审团从哪来:整个实验只在阶段 1 生成过一次 rollout(每题 256 条,共 8×256 = 2,048 条), 之后再没有任何新采样。D2 的每个陪审团都是从本题这 256 条池子里重抽的子集: 对每个大小 m,独立重复 32 次"不放回抽 m 条"。"不放回"只约束单个陪审团内部 (同一陪审团里不会有重复的 rollout,模拟训练组);陪审团之间独立重抽,同一条 rollout 可进多个陪审团。 于是一道题在 m=64 档共有 32×64 = 2,048 个成员席位,由 256 条去坐——平均每条坐 8 个席位, 这就是下表"期望被抽进 32·m/256 个陪审团"的含义。策略冻结 ⇒ 从池子抽 m 条与新采 m 条分布等价; 而复用同一批 token 是设计的前提——参考真值 ι* 只对这批 token 算过,考新 token 就没有标准答案可对。 副作用仅有陪审团间共享成员带来的相关性,它影响 32 团平均的误差条,不影响单团估计的无偏性。 代码:run_d2_stage 外层 for k in range(32),内层 members = rng.choice(R, size=m, replace=False)。

在此基础上,分清两个"token 数":

证据侧(定一个符号用了多少证据)考卷侧(哪些 token 被打分)
m = 8其余 7 条 ≈ 2,800 个 token 同一批:每题 R=256 条 rollout 的全部 token。每条 rollout 期望被抽进 32·m/256 个陪审团(m=8 约 1 次;m=64 约 8 次)
m = 64其余 63 条 ≈ 25,000 个 token
  • 证据侧是实验的自变量:准确率随 m 单调上升,说的就是"证据越多、符号越准"——欠采样结论的直接来源;
  • 考卷侧分布不变:报告表里 token 数从 ~51 万涨到 ~421 万,是打分事件数(token × 陪审团),不是换了考题——大 m 只是把同一批 token 重复评了 ≈8 次,降方差、不引偏;
  • 逐 token 的质量权重 |d·ι*| 与参考答案(R=256 + 留一)对所有 m 是同一份;
  • 唯一的小选择效应:退化陪审团被跳过(m=8 时约 2–4%,报告中列出),同一 rollout 在其他陪审团照样被评,可忽略;
  • 忠实性约定:训练里 m=8 的组只给自己组员的 token 定符号,探针照做(陪审团只给成员打分;考卷覆盖靠每档 32 次重采样);另一种设计——让小陪审团给全部 256 条打分——统计上更省,但不再是训练里发生的事;
  • 内证:D3 的 acc1 与 D2 的 m=8 行几乎相同(DeepMath 58.6% ↔ 58.7%;GSM8K 63.9% ↔ 64.5%)——同一估计量在两个独立重采样方案下读数一致。

5各阶段详解:存什么文件、算什么

阶段 1 · rollout(vLLM 采样)

算:先对 64 道候选题各采 16 条粗筛出通过率,选最接近 0.5 的 8 道;再对每道选中题采 256 条(温度 1.0、上限 1024),逐条判分。

文件内容
screen.json64 道候选题的粗筛通过率(题目筛选的证据)
questions.json选中 8 题:粗筛/主采样通过率、截断率(应 <5%)、prompt token ids
q*/rollouts.jsonl每行一条:completion_ids(token 序列)、verdict(0/1)、truncated、原文
manifest.json / versions.json解析后的完整配置与包版本(复现凭证)

阶段 2 · teacher(教师前向)

算:教师对每条 rollout 前向一次,取实际采到 token 上的 log πT。 存:q*/teacher_logp.npz(每条 rollout 一个长度 T 的数组)。之后教师即卸载。

阶段 3 · reference(参考真值)——最重要的一张表

算:全部 256 条按式 (1)(3) 建 Gref;同时按奇偶建 GA、GB(128 条各半,自检用)。 再对每个 token 按式 (4)(6) 打分得到参考影响力 ι*,以及两半各自给的符号。 存:q*/tokens.npz,每行一个 token:

列含义来自
rollout, postoken 的地址(第几条、第几个)——所有后续表用它对齐—
ytoken id采样
d教师信号 dt,式 (2)阶段 2 − 本阶段学生 logP
ref_iota参考影响力 ι*:R=256 大陪审团 + 精确留一,式 (4)(6)本阶段
signA, signB参考两半(128/128)各自的符号——参考自稳定性检验用本阶段
ttypetoken 类型:数字 / 符号 / 单词 / 其他(分解分析用)分词器解码
trunc, verdict所属 rollout 是否截断、是否判对阶段 1

阶段 4 · d2(陪审团大小扫描)

算:对每个大小 m ∈ {8, 16, 32, 64}(报告列名 G),每道题独立重采 32 个陪审团(不放回)。全对/全错的陪审团记为退化、跳过。 对每个非退化陪审团,完全按训练算法算出其成员 token 的:

  • iota_hat = 全陪审团 + 精确留一的估计(训练里的 iota,式 (4)(6) 用小 G);
  • alt_sign = 对面半批的符号(训练里的 iota_alt,§4.6)。

存:q*/d2.npz(size, jury, rollout, pos, iota_hat, alt_sign)+ d2_juries.json(每个陪审团:是否退化、组内通过率)。跨 m 比较的公平性分析见 §4.8。

阶段 5 · d3(三票对照)

算:模拟 200 个训练组(m=8),每个 token 三票——

  • v1 全组留一(训练的 iota;陪审团 = 其余 7 条);
  • v2 对面半批(训练的 iota_alt;陪审团 = 对面 4 条);
  • v3 本半批去己(陪审团 = 本半另外 3 条)——与 v2 的陪审团不相交,这才是真独立的两票。

关键:v1 的陪审团(7 条)完整包含 v2 的陪审团(4 条),所以训练的 agree(v1 vs v2)天然偏高;v3 vs v2 才是干净的分半信度。 存:q*/d3.npz(group, rollout, pos, v1, v2, v3)。

阶段 6 · analyze(汇总)

存:results/summary.json(全部数字)+ results/report.md(中文报告,预写判读已嵌入)。公式见下节。

6最终指标:每个数怎么算

6.1 质量加权符号准确率(D2 的主指标)

把小陪审团的符号 ŝt = sign(iota_hat) 与参考符号 s*t = sign(ref_iota) 逐 token 对比(双方非零才计入):

accw = Σt wt · 𝟙{ŝt = s*t} / Σt wt , wt = |dt · ι*t|
(7)

同时报不加权与 |ι*| 加权两个变体。为什么用 |d·ι| 当权重:它就是论文里的"质量"——命题 B.2 说带估计符号的实际一阶增益 = 判对质量 − 判错质量,于是有一个精确换算:

实际一阶增益 = (2·accw − 1) · Σt |dt ι*t|
(8)

所以 accw = 0.5 ⇔ 方法的可证优势被符号误差完全抵消;accw 每高出 0.5 一分,优势就按比例兑现一分。这是"符号准确率"与论文定理之间的直接桥梁。

6.2 门控指标(训练机制值不值)

门控保留率 = 两票一致(ŝt = alt_sign ≠ 0)的 token 占比;门控后准确率 = 在保留子集上重算式 (7)。 门控的价值 = (门控后准确率 − 门控前) × 保留率:提升大、保留高才值得。

6.3 校准与分解

  • 十分位校准:按 |iota_hat| 从小到大分十档,每档算式 (7)——检验"估计越自信是否越准"(理想应单调上升;若平坦 = 幅度不携带信息);
  • token 类型分解:数字 / 符号 / 单词 / 其他各算一个 acc——看符号到底在哪类 token 上有信号(若只在格式类 token 上准,方向就是在放大 F8 的噪声);
  • 退化率:每档 G 的退化陪审团占比(G 越小越容易全对/全错——targeted 在退化组自动变回 OPD 的量化)。

6.4 参考自稳定性(真值自己可不可信)

把 256 条拆成不相交的两半(各 128),各自独立算符号,按式 (7) 的权重算一致率。 两半完全独立,若各以概率 p 命中"无穷陪审团"的真符号:

agree = p2 + (1−p)2 ⇒ p = (1+√(2·agree−1))/2
(9)

报告里设了 85% 的门槛:低于它就提示"参考本身欠稳,绝对数值保守解读(相对比较仍有效),可加大 R"。

9常见问题

为什么温度必须 1.0,不能用贪心或降温?

符号刻画的是"当前策略"的性质,采样分布必须等于该策略的自然分布(τ=1 是随机采样,贪心是 τ→0)。 降温采样 = 从另一个分布取样,估的就不是训练时那个对象了;而且贪心会让同题 256 条完全一样,陪审团直接退化。

为什么一定要挑"有对有错"的题?

所有信号都从优势 A = V − 组均值来。全对或全错 ⇒ A≡0 ⇒ G≡0 ⇒ 没有任何符号。粗筛选通过率≈0.5 的题让退化陪审团尽量少 (退化率本身也是 D2 的输出之一——它量化了训练里"targeted 悄悄变回 OPD"的比例)。

参考真值凭什么算"真"?

它不是绝对真值,是 R=256 的大样本估计。所以报告第一项就是自检:把 256 拆成两个独立的 128,按式 (9) 看一致率并反解出单半准确率 p。 两半一致率 ≥85% 才继续;不够就加大 R。所有 D2/D3 结论都是"相对这个参考"的,报告里如实标注。

为什么主指标用 |d·ι| 加权,而不是每个 token 一票?

因为训练里 token 的奖励幅度是 |d|、理论增益是 |d·ι| 的质量:符号在高质量 token 上判错的代价远大于低质量 token。 式 (8) 说明质量加权准确率可以直接换算成方法每步实际兑现的一阶增益——这是与论文命题 B.2 严格对应的口径。不加权版本也同时报告。

D3 为什么要第三票?两票不够吗?

训练的两票(全组留一 vs 对面半批)共享数据:前者的陪审团(7 条)完整包含后者的(4 条),一致率天然虚高。 第三票"本半批去己"(3 条)与对面半批完全不相交,v3-vs-v2 才是干净的分半信度;差值 inflation 就是训练仪表 0.76 里的水分。

陪审团大小 m 变大时,表里的 token 数涨到几百万——比较还公平吗?

公平。首先,陪审团不是新生成的 rollout——全程只有阶段 1 采过的每题 256 条,陪审团是对这个固定池子的重抽子集(团内不放回、团间独立)。其次,表中 token 数是"打分事件数"(token × 陪审团),不是不同的 token 集合:所有 m 档打分的都是同一批每题 R=256 条 rollout 的 token,每条 rollout 期望被抽进 32·m/256 个陪审团(m=8 约 1 次、m=64 约 8 次)。大 m 只是重复评分,降方差不引偏;真正被操纵的自变量是证据侧——每个符号背后其余 m−1 条的 token 量。详见 §4.8。

这个实验为什么不用训练就能回答 F1?

符号质量是"当前策略 + 估计算法"的性质,不是训练过程的性质。冻结策略下把估计算法原样搬来、对着大样本参考打分即可。 训练只会改变策略从而改变符号质量的数值(所以第二步建议再探一个训练中期检查点),但"G=8 下符号可不可信"在初始策略上就能裁决。
