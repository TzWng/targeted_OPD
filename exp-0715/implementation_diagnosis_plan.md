# Targeted-OPD 代码重建与逐层诊断计划

> 版本:v1(2026-07-15)
> 适用:基于 exp-0715 两次战役(GSM8K 07-12、DeepMath 07-13)的复盘结论,重写一版代码,并在重写过程中逐层定位"结果不理想"的真正原因。
> 配套仪表盘(交互版复盘,含全部证据):https://claude.ai/code/artifact/c2ad998b-2376-4179-9b55-855848671728

---

## 0. 总原则

1. **一层一验收**:每个模块搭完先过"验收标准"再往上搭。验收不过,不写下一层。
2. **一次只改一个东西**:每次运行相对一个命名的父运行恰好差一个旋钮(旧 asym 版一次改了 ≥4 个,归因失败)。
3. **先测量后训练**:凡是能用推理(不训练)回答的问题,一律先用推理回答。本计划的 A、D 两个阶段几乎不花训练算力。
4. **机器可读**:每步一行 JSONL;运行开始打印完整解析后的 Config 并存盘(旧版日志头与文件默认值矛盾:32 vs 64 rollouts/步、cap 768 vs 512,实际配置已不可考——这次不能再发生)。
5. **固定探针集**:随机抽 32 道题(难度带内,种子固定),缓存基座模型每题 8 条 rollout。所有"快检"都在它上面做,分钟级出结果。
6. **噪声预算贴在墙上**:n=100→±5.0pp,n=200→±3.5pp,n=500→±2.2pp,n=1000→±1.6pp(1σ)。**单次非配对评测,|Δ|<2σ 一律不当结论。**

### 要定位的十个疑点(来自复盘,按优先级)

| 编号 | 疑点 | 覆盖它的测试 |
|---|---|---|
| F1 | 影响力符号 ≈ 抛硬币?(mis≈0.46,agree≈0.76,从未对照真值) | **D2–D4**(核心) |
| F2 | 沙盒推不动任何方法(OPD n=1000 仅 +0.6pp) | **C1–C3** |

| F7 | 指标语义误导(acc_in_batch 非正确率;NaN 被静默丢弃) | F1(新指标定义) |

## 阶段 A:地基——验证器、数据、评测协议(不碰模型,~半天,CPU 为主)

### A1. 验证器单元测试
- **做法**:新建 `tests/test_verifier.py`,覆盖 ≥20 个已知对错的 (pred, gold) 用例:
  嵌套 `\boxed{\frac{1}{2}}`、`\dfrac` vs `\frac`、带逗号数字 `1,234`、`0.5` vs `\frac12`、
  截断导致花括号不闭合、`#### 42` 格式、末尾多句号、区间/集合答案、负数、
  math-verify 抛异常时的回退路径。
- **验收**:全部用例通过;同时统计——从旧日志随机抽 50 条被判错的 rollout,人工复核误判率。
- **失败含义**:若误判率高,F3 的"标签噪声"一部分根本不是截断而是解析,先修这里。

### A2. 数据加载检查
- **做法**:加载 DeepMath 难度 (0,3.0],打印题数、字段样例、难度直方图;GSM8K 同理。
- **验收**:DeepMath n=3,434 与旧日志一致;随机抽 5 题人工看 gold 格式。

### A3. 冻结评测协议(对应 R0.2)
- **做法**:写独立的 `eval.py`(贪心、cap 1024、n 可配、同一套 prompt 顺序),然后做三个一致性检查:
  1. 同一模型同配置**连续评测两次**(vLLM 贪心应逐字一致);
  2. **vLLM 路径 vs HF 路径**各评一次;
  3. 基座模型复现旧日志基线:GSM8K test[:100] ≈ 0.470,MATH-500[:200] ≈ 0.315。
- **验收**:① 两次差异 = 0;② 两路径差异 < 1pp;③ 基线复现差异 < 1pp。
- **失败含义**:①不为零→解码配置有非确定性;②大→旧结果里 vLLM 热插拔可能没换上正确的 adapter(严重!);③大→环境/版本变了,老结论作废需注明。
- **附带产出**:跑一次 **R0.1——把旧检查点全部复评**(六组运行 × 每 100 步的快照,GSM8K n=1000、MATH 全 500)。这一步零训练,直接裁定"第 150 步的冲高是否真实"(F5、F6)。

## 阶段 B:采样与训练回路的"哑测试"(~1 天,少量 GPU)

### B1. rollout 管道体检
- **做法**:探针集上采样,记录 len 分布、cap 命中率、EOS 率、空回答率;训练 cap 先改成 1024(与评测一致)。
- **验收**:cap 命中率 < 10%(旧版 DeepMath 是 ~45%,这是 F3 的直接修复);每组 k/G 直方图落盘。

### B2. 策略一致性检查(容易被忽略、一旦错全盘皆输)
- **做法**:同一批 rollout 的 token,比较「vLLM 采样引擎(热插拔 adapter 后)」与「HF 训练前向」给出的 logP;
  另:故意更新一步 LoRA 后重复,确认 vLLM 拿到的是**新**权重。
- **验收**:|ΔlogP| 中位数在 bf16 容差内(~1e-2 量级);更新后 vLLM 输出随之变化。
- **失败含义**:训练在优化一个和采样不同的策略——任何 RL 结论都无效。旧代码逻辑上做了热插拔(`_fresh_adapter_request`),但从未验证过数值一致性。

### B3. 零学习率烟测
- lr=0 跑 20 步:pass@1 不变、无 NaN、显存曲线平稳。**验收**:全部满足。

### B4. 过拟合金丝雀(策略梯度正确性的经典测试)
- **做法**:取 4 道探针题,用 GRPO(mode=grpo)只在这 4 题上反复训练 ~100 步。
- **验收**:这 4 题的组内正确率 → 接近 1.0。
- **失败含义**:REINFORCE 损失、completion mask(EOS 计入、pad 排除、prompt 排除)、优势计算或梯度累积有 bug。这是最便宜的"实现正确性"证明,旧代码从没做过。

### B5. 教师信号体检(F8)
- **做法**:探针集 rollout 上计算 d_t,分别统计「数学内容 token」与「格式 token(####、\boxed、换行等)」两组的 d_t 分布;
  同时用 Qwen2.5-Math-1.5B 和 Qwen2.5-1.5B-Instruct 两个教师各算一遍。
- **验收/判读**:若 Math 教师在格式 token 上出现大幅负 d_t(教师"讨厌"学生的作答格式),则 GSM8K 战役的教师信号有相当部分花在文风上——GSM8K 换 Instruct 教师,或统一 \boxed 格式。

## 阶段 C:基线先行——先让一个方法明确地涨(每项 ~1 GPU 天)

> 过关闸门:**至少一个方法在冻结协议下比基线高 ≥2σ**(n=1000 时约 +3pp)。闸门不过,后面 D、E 全部暂停——因为沙盒本身检测不到学习(F2)。

### C1. GRPO 校准跑(对应 R1.1)
- 150 步短跑扫 lr {1e-5, 3e-5, 1e-4} × LoRA r {16, 64},赢家延长到 800 步;盯 nondegen、长度漂移、熵。
- **验收**:≥2σ。**失败含义**:调 lr/秩/难度带/cap,而不是怀疑蒸馏方法。

### C2. KL 形式的 OPD(GKD 式,对应 R1.2)
- 用重建版 targeted_kl 关掉倾斜(a≡1)= 全词表闭式 KL 蒸馏。同预算。
- **验收**:出现文献量级的正增益(该师生对差距 ~35pp,OPD 理应可见)。

### C3. REINFORCE 形式的 OPD(旧代码的 opd 模式)
- 与 C2 同预算对比。**判读**:若 C3 明显不如 C2 → 旧战役全用了高方差路线,这本身就是"结果平"的部分解释;后续所有蒸馏一律走 KL 路线。

## 阶段 D:影响力估计器隔离检验(核心阶段,几乎全是推理,~1–2 天)

> 这是 F1 的审判庭。全部在**冻结 checkpoint**上做,与训练解耦。

### D1. 数值单元测试
- **做法**:`tests/test_influence.py`——
  ① 小词表(V=50、h=8)随机数据上,`compute_influence` 的 G、留一修正、matvec 恒等式 ι̂(y)=(Gh)_y−E_π[Gh] 逐项对照暴力循环实现,误差 <1e-5;
  ② 论文附录"两路径"小例子(命题 3 的构造)手算对照;
  ③ 留一的自项对角 = 附录 B 的偏差项(旧代码注释声称验证过,重建版进 pytest 固化)。
- **验收**:pytest 全绿。

### D2. 符号准确率 vs 陪审团大小(**信息量最大的单个实验**,对应 R2.1)
- **做法**:8 道探针题,每题采 256 条 rollout → 大陪审团 ĝ_Q^ref 与参考符号;
  再从中不放回抽 G ∈ {8, 16, 32, 64} 的小陪审团,按训练时的完整流程(留一、门控)估符号;
  汇报「按 |d·ι| 质量加权的符号一致率」——整体、按 |ι̂| 十分位、按 token 类型。
- **验收/判读**(写死在报告里):
  - G=8 时一致率 **< 60%** → F1 实锤:现行配置下符号≈噪声,**先修估计器再谈任何 targeted 战役**;
  - 曲线在 G=32 处明显抬升 → 解法明确:定符号用大 G(只采样不训练);
  - 各 G 都上不去 → 换投影子空间(D4)或承认最后一层信号不足。

### D3. 分半一致率的水分定量
- **做法**:三种"两票"配置各算 agree:
  ① 现状:全批留一 vs 对面半批(两票共享对面半批数据,相关);
  ② 真独立:本半批去己 vs 对面半批(陪审团不相交);
  ③ 各票 vs D2 的参考真值。
- **判读**:agree(①) − agree(②) 就是 0.76 里的"相关性水分";agree(③) 才是真准确率。一行代码的改动,直接回答"0.76 可信吗"。

### D4. 投影子空间对照
- lm_head 块(现状,LoRA 恰好不训练它)vs LoRA 块 vs 随机草图,各自对 D2 参考真值的一致率。
- **判读**:若 LoRA 块显著更准 → 换投影(符号应活在被训练的子空间里)。

### D5. 截断污染对符号的影响
- 含 / 不含 capped rollouts 建陪审团,符号翻转率。**判读**:量化 F3 → F1 的传导。

### D6. 语义抽查(给直觉,防自欺)
- 取被翻转质量最大的 20 个 token,人工看:上下文、教师/学生概率、翻转方向。
- **判读**:翻的是"数学步骤"还是"格式/连接词"?若全是格式 token,说明符号在放大 F8 的噪声。

## 阶段 E:targeted 的分步组装(单旋钮消融,~2–3 GPU 天)

> 前提:C 闸门已过、D2 给出的符号准确率可接受(或已按 D 的结论修好估计器)。
> 全部在**同一个新框架**里以开关实现(F10 的教训:不再 fork 文件)。每臂 2 个种子、同预算、冻结协议、汇报"最终 + 选峰 + 曲线下面积"。

| 臂 | 配置 | 相对上一臂只多一件事 | 判读 |
|---|---|---|---|
| E0 | KL-OPD(a≡1,= C2) | — | 基准 |
| E1 | + 门控,但确认反向的 token 只**置零**(论文原版中性桶 r=0) | 只"闭嘴"不"反向" | E1 < E0 → 门控在乱闭嘴(误伤好 token) |
| E2 | + 温和反向 amp_flip=0.25 | 开始反推 | E2 < E1 → 反向方向多半是错的(符号差) |
| E3 | + 全强度反向 amp_flip=1(论文 eq. 12) | 反推加满 | E3 vs E2 的差 = 符号质量的杠杆 |
| E4 | + 同向放大 amp_same=1.25(asym 完全体) | 外推预算 | 单独归因 gopd 式外推的贡献 |
| E5 | E3 但定符号用 G=32 大陪审团(按 D2 结论) | 估计器升级 | 直接检验"符号准了结果就好"的因果 |

- **判读逻辑**:从 E0 → E5 哪一臂开始掉,问题就在那一步引入的组件上。整表就是 F1/F10 的最终归因。
- 每臂同时记录 F 阶段定义的全部仪表(尤其 predicted vs realized gain)。

## 阶段 F:仪表与汇报(重建时一并写好,不另花时间)

1. **每步 JSONL**:predicted_gain = Ê[Σ|d||ι̂|]、margin = 2Σ_{d·ι<0}|d·ι|(论文 §2.4 本来就要求汇报)、agree(三种口径)、mis、nondegen、cap、len、**真实批正确率**、k/G 直方图、熵、grad-norm、lr。
2. **改名**:`acc_in_batch` → `frac_pos_adv`(F7);NaN 显式落盘,不允许静默剔除。
3. **每 50 步**:固定探针集上的 ΔQ → "预测增益 vs 实际增益"对照曲线(方法是否兑现理论的直接读数)。
4. **每 100 步存 checkpoint(全保留)**;训练结束自动触发 n≥500 复评脚本,产出带置信区间的快照表。

## 建议的代码结构

```
opd_v2/
  config.py        # dataclass;运行时打印+存盘解析后配置(F9)
  verifier.py      # 判分(A1 的被测对象)
  data.py          # 加载与难度带(A2)
  rollout.py       # vLLM 采样 + len/cap 统计(B1、B2)
  teacher.py       # 教师前向、d_t(B5)
  influence.py     # G / 留一 / matvec / 门控(D1 的被测对象)
  rewards.py       # 七种模式 + E0–E5 的开关,全部纯函数
  train.py         # 主循环(梯度累积、调度、JSONL)
  eval.py          # 冻结协议(A3);snapshot_eval.py 批量复评
  probes/          # D2–D6 独立脚本,只依赖冻结 checkpoint
  tests/           # pytest:verifier / influence / mask / 优势
```

- 关键差异 vs 旧版:奖励全部是**纯函数**(输入 d_t、ι̂、A、门控配置 → r_t),单测友好;probes 与训练解耦;不再按变体 fork 文件。

## 预算与顺序(参考:旧配置 ~125 s/步,800 步 ≈ 27 GPU·时/臂)

| 周 | 内容 | 算力 |
|---|---|---|
| 第 1 周 | A 全部 + R0.1 旧检查点复评 + B 全部 | < 1 GPU 天 |
| 第 2 周 | C1–C3(闸门)+ D1–D6 | ~3–4 GPU 天 |
| 第 3 周起 | E0–E5(仅当 C、D 都过) | ~4–6 GPU 天 |

## 最可能的三种结局(提前写下,防事后合理化)

1. **D2 显示 G=8 符号≈噪声** → 结论:想法未被否定,是估计器欠采样;走 E5 路线(大陪审团定符号)或 D4 换投影,论文实验章相应改写。
2. **C 闸门都过不了** → 结论:0.5B+LoRA+此预算的沙盒太弱;换更强训练配置或更大学生,旧的"方法比较"全部作废重跑。
3. **符号准(>70%)但 E3 仍不赢 E0** → 结论:一阶增益没有兑现,查步长/裁剪/漂移(F6),或触发论文自己给出的反例条件(师生解法不同,transfer 条件失效)——这将是论文层面最有价值的发现。
